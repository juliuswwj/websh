📱 用手机和 Claude 对话

websh 使用说明 · 面向不熟悉 Linux / x-workbench 的同学

websh 是一个装在手机浏览器里的终端 App。登录后你可以在手机上打开服务器里的 Claude（claude-code）会话，随时随地继续对话。会话跑在服务器的 tmux 里， 手机断网、息屏、切后台都不会丢——重新打开会自动接回原来的对话。

🚫 最重要的一点：公司内网打不开，换个网络就行

下面这个网址在公司内网里访问不到：

https://ai.rainier.lango-tech.com/websh/

打不开通常是连了 Lango-5G、Lango-GMS 这类公司内网 Wi-Fi。 只要换成非内网的网络就能打开——比如 Lango-VIP、你家里的宽带，或者手机流量都行。 在公司内网怎么刷都出不来是正常的，不是坏了。

一、首次配置（在服务器上做一次）

第一次使用前，需要在你的服务器账号里建一个配置文件。这一步要登录服务器操作一次， 之后手机端就一直能用了。如果你对命令行不熟，可以照抄下面的命令，或找运维/同事帮忙跑一遍。

🔌 这一步要在公司内网里做（和手机端正好相反）

配置需要 SSH 登录服务器，必须在公司内网（包括有线网络）下操作。 在电脑的终端里运行下面命令登录（把 user 换成你的服务器账号）：

ssh user@192.168.21.240

记住：配置在公司内网做、手机日常用在非内网做，两边要求的网络正好相反。

  1. 生成登录验证码（TOTP）

    连上服务器后，运行：

    websh config

    它会在终端里打印一个二维码。用手机上的身份验证器 App （Google Authenticator、Microsoft Authenticator、1Password 等）扫这个码， 之后 App 里会每 30 秒刷新一个 6 位数字——这就是登录用的「动态验证码」。

  2. 填入你的 x-workbench 账号，加上要用的服务器

    编辑配置文件 ~/.config/websh.yaml（命令：nano ~/.config/websh.yaml）， 在里面补上 xwb: 段和一台 type: xwb 的服务器。改成你自己的信息：

    otp_secret: "……"        # 上一步自动生成的，别动

# 你的 x-workbench 账号（websh 用它登录 xwb 拿到令牌）
xwb:
  host: "172.60.1.35:9630"     # x-workbench 服务地址（按实际填写）
  email: "you@lango-tech.com"  # 你的 x-workbench 邮箱
  password: "你的密码"

# 你要在上面跑 Claude 的服务器，用它在 x-workbench 里的 IP 标识
remotes:
  - type: xwb
    ip: "10.0.0.5"             # 服务器在 x-workbench 里的 IP
    name: "我的开发机"          # 随便起个好认的名字（可选）
    💡 不知道 host / ip 填什么？

    host 是 x-workbench 的服务地址，ip 是你平时用的那台开发机 在 x-workbench 里的地址。如果不确定，问一下负责 x-workbench 的同事，或参照你 Web 版 x-workbench 里看到的服务器。 server_id / credential_id 这些 websh 会自动帮你查，不用填。

二、在手机上打开并登录

  1. 换到非公司内网，打开网址

    只要不是 Lango-5G / Lango-GMS 这类公司内网（Lango-VIP、家里宽带、手机流量都行）， 浏览器打开 https://ai.rainier.lango-tech.com/websh/。

  2. 装到主屏幕（推荐）

    iPhone：Safari 点「分享 → 添加到主屏幕」；Android：Chrome 菜单「添加到主屏幕 / 安装应用」。 装成 App 后体验更好，而且开启消息通知必须先装到主屏幕。

  3. 登录

    填三样：用户名（服务器账号）、密码（服务器账号密码）、 6 位验证码（身份验证器 App 里那串数字）。

三、打开 Claude 开始对话

  1. 点左上角 ≡ 会话 打开会话列表。
  2. 找到你配的那台服务器，点它的 ＋ …… · Claude 按钮，开一个 Claude 会话。 （旁边的 ＋ …… · Bash 是普通命令行终端。）
  3. 等它连上后，就能像在电脑上一样和 claude-code 对话了。

怎么打字、怎么发送

底部有两种输入方式，点最左边的按钮切换：

  • ✍️ 整句：在底部输入框里打整句，点「发送」或回车，一次性发给 Claude。推荐。
  • ⌨️ 单字：调出手机软键盘逐字输入，适合需要按方向键、Esc、Ctrl 组合键的场景 （上面一排有 Esc / Tab / ^C 等快捷键）。

回车即发送：直接按回车就会把当前内容提交给 Claude（不会变成换行）。

手势操作（直接在终端画面上）

手势作用
单指上下滑像滚轮一样翻看历史输出
单指左右滑切换到上一个 / 下一个会话
长按后拖动框选一段文字（松手即复制，需 HTTPS）
翻到历史后轻点回到最新位置
两指捏合放大/缩小字号

底部按钮：📋 复制面板 · 📄 复制整屏 · 📥 粘贴（手机版的 Ctrl+Shift+V）· 🧹 清屏 · 🔄 重连。

四、开启消息通知（强烈推荐）

Claude 跑完一段、需要你确认或回复时会「响铃」。开启通知后，即使你切到别的 App 或锁屏， 手机也会弹推送提醒你，不用一直盯着。

  1. 确保已经装到主屏幕并从主屏幕图标打开（直接在浏览器里开不行）。
  2. 打开 ≡ 会话 抽屉，点 🔔 开启通知，允许权限即可。
⚠️ iPhone 用户注意

iOS 需要 16.4 以上系统，并且必须先「添加到主屏幕」、再从那个图标进去开启通知， 在 Safari 里直接开是不行的。Android Chrome 没有这个限制。

常见问题

网址怎么都打不开 / 一直转圈

八成是连着公司内网 Wi-Fi（Lango-5G、Lango-GMS 等）。换成非内网的网络再试—— Lango-VIP、家里宽带、手机流量都行。

登录失败 / 验证码不对

验证码每 30 秒变一次，看准当前那一组再输；用户名密码是服务器账号的，不是 x-workbench 的。 如果手机时间不准也会导致验证码对不上，确认手机用「自动设置时间」。

会话列表里没有 Claude 按钮

说明 ~/.config/websh.yaml 里还没配 xwb: 段或没加 type: xwb 的服务器。 回到「一、首次配置」第 2 步检查。

打开 Claude 时报错（连不上 / 找不到服务器）

多半是 host、ip 或 x-workbench 邮箱/密码填错了，或者那台服务器在 x-workbench 里 用的不是这个 IP。核对配置，必要时找 x-workbench 管理员确认地址。

中途断网了会不会丢对话？

不会。Claude 在服务器上继续跑，websh 会自动重连，回来后接着看。这正是用 websh 的好处。

怎么把 Claude 的回答复制出来？

长按拖动框选要的文字（松手即复制）；或点 📄 复制整屏、 📋 打开复制面板手动选。复制到剪贴板需要 HTTPS，本站已是 HTTPS，没问题。
